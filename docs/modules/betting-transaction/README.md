# 投注交易模块完整清单 (修订版)

## 1. 修订概述

本文档是根据您的最新反馈修订的版本。主要变更包括：

- **组件移除**：移除了因缺少数据源而暂时不需要的 `赛事统计` 和 `历史交锋` 组件。
- **组件整合**：将各类状态提示（如空状态、赔率变化、盘口关闭）的功能统一整合到新增的两个全局组件 `模态消息 (Modal)` 和 `Toast` 中，使提示方式更标准化。
- **投注栏重构**：对核心的 `投注栏` 组件进行了重新设计，引入了 `Slip` (注单)、`Open` (待结算)、`Settled` (已结算) 三个Tab，将投注历史功能初步整合进组件内部，从而替代了原有的独立“投注历史页”。

这些调整使得组件清单更加精简和贴近当前实际需求。

## 5. API 清单 (APIs)

API 清单也根据组件的调整进行了更新。

| 序号 | API 名称 | 推测的 ID | 功能描述 |
| :---: | :--- | :--- | :--- |
| 1 | **提交投注** | `place-bet` | 接收用户的投注单数据，进行验证后提交到 MTS 系统。 |
| 2 | **获取用户余额** | `get-user-balance` | 查询用户当前的账户余额，用于投注前的余额检查。 |
| 3 | **验证投注** | `validate-bet` | 在用户提交投注前，验证投注单的有效性（赔率、盘口状态）。 |
| 4 | **获取赛事列表** | `get-event-list` | 根据筛选条件获取赛事列表。 |
| 5 | **获取赛事详情** | `get-event-detail` | 根据赛事 ID 获取单场比赛的详细信息和盘口数据。 |
| 6 | **获取待结算注单** | `get-open-bets` | **(新增)** 获取在 `投注栏` > `Open` Tab 中显示的待结算注单列表。 |
| 7 | **获取已结算注单** | `get-settled-bets` | **(新增)** 获取在 `投注栏` > `Settled` Tab 中显示的已结算注单列表。 |
