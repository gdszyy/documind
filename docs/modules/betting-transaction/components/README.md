# 投注交易模块 - 组件文档

本目录包含投注交易模块的所有组件文档，所有文档均按照 **component-template-v6.md** 规范编写。

## 📋 组件清单

### 全局组件 (Global Components)

全局组件是跨页面、管理复杂状态的共享组件，通常在应用中是单例。

| 组件名称 | 文件 | 描述 | 状态 |
|---------|------|------|------|
| 投注栏 | [bet-slip.md](./bet-slip.md) | 管理用户的注单选择、待结算和已结算历史，包含三个Tab（Slip/Open/Settled） | draft |
| 顶部导航栏 | [navbar.md](./navbar.md) | 提供站点级别的核心功能导航，并展示用户状态信息 | draft |
| 体育/联赛导航 | [league-navigation.md](./league-navigation.md) | 以可折叠的树状结构展示所有体育项目和联赛 | draft |
| 底部导航栏 | [bottom-nav.md](./bottom-nav.md) | 在移动端设备上提供核心功能的快捷访问入口 | draft |
| 模态消息 | [modal-message.md](./modal-message.md) | 用于显示需要用户明确响应的关键信息、确认操作或警告 | draft |
| Toast | [toast.md](./toast.md) | 用于显示非侵入式的、短暂的通知消息 | draft |

### 普通组件 (Normal Components)

普通组件是构成页面的基础UI单元，通常是无状态或状态简单的可复用组件。

| 组件名称 | 文件 | 描述 | 状态 |
|---------|------|------|------|
| 赛事卡片 | [event-card.md](./event-card.md) | 在赛事列表页中，以卡片形式展示单场比赛的核心信息和主要盘口 | draft |
| 赛事信息 | [event-info.md](./event-info.md) | 在赛事详情页顶部，展示该场比赛的详细信息 | draft |
| 盘口列表 | [odds-list.md](./odds-list.md) | 在赛事详情页中，按盘口类型组织和展示所有可投注的盘口 | draft |

## 📐 文档规范

所有组件文档遵循以下章节结构：

1. **概述** - 组件功能和用途的详细描述
2. **视觉预览** - 组件的默认状态预览图
3. **组件剖析** - 组件内部元素的结构说明
4. **内容指南** - 各元素的内容要求和示例
5. **属性与状态** - 状态维度、状态矩阵和交互状态
6. **行为与交互** - 用户操作和条件逻辑规则
7. **使用示例** - 代码示例
8. **无障碍设计** - 键盘导航和屏幕阅读器支持
9. **技术规格** - 关联关系、Props、技术细节和CSS变量
10. **变更历史** - 版本变更记录
