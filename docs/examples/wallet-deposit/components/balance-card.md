# 余额卡片

## 1. 概述

余额卡片用于显示用户的当前钱包余额，并提供刷新功能。

### 使用场景

- 充值页面顶部显示当前余额
- 钱包首页显示余额概览

### 不使用场景

- 不用于显示其他类型的金额（如订单金额）

## 2. 视觉预览

```
┌─────────────────────────────────┐
│  我的余额                        │
│                                 │
│  ¥ 1,234.56                    │
│                                 │
│  [刷新图标]                      │
└─────────────────────────────────┘
```

## 3. 组件剖析

| 元素 | 功能 | 是否必需 |
|------|------|---------|
| 标题文本 | 显示"我的余额" | 是 |
| 余额数字 | 显示当前余额，格式化为货币 | 是 |
| 刷新按钮 | 点击刷新余额 | 是 |

## 4. 内容指南

- **余额数字**: 使用货币格式，保留 2 位小数，千分位分隔符
- **刷新按钮**: 使用图标，无文字

## 5. 属性与状态

### 5.1. 状态维度

| 维度 | 可选值 |
|------|--------|
| 加载状态 | loading, loaded, error |

### 5.2. 状态矩阵

| 状态 | 余额数字 | 刷新按钮 | 说明 |
|------|---------|---------|------|
| loading | 显示骨架屏 | 禁用 | 正在加载余额 |
| loaded | 显示实际余额 | 启用 | 加载完成 |
| error | 显示"加载失败" | 启用 | 加载失败 |

## 6. 行为与交互

### 6.1. 用户操作

| 操作 | 触发条件 | 结果 |
|------|---------|------|
| 点击刷新按钮 | 用户点击 | 重新调用查询余额 API |

## 7. 使用示例

```jsx
<BalanceCard 
  balance={1234.56}
  loading={false}
  onRefresh={() => fetchBalance()}
/>
```

## 8. 无障碍设计

- 刷新按钮应有 `aria-label="刷新余额"`
- 余额数字应有 `role="status"` 属性

## 9. 技术规格

### 9.1. 关联关系

- 被使用于: [@充值页面](../pages/deposit-page.md)
- 调用 API: [@查询余额 API](../backend/apis/get-balance.md)

### 9.2. 组件 Props

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| balance | Number | 是 | - | 余额金额 |
| loading | Boolean | 否 | false | 是否正在加载 |
| onRefresh | Function | 是 | - | 刷新回调函数 |
